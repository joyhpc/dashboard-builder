# 案例：全球城市气温仪表盘

**适合场景**：数据有 时间 + 多个地点 + 一个连续数值指标，且有季节性或长期趋势。比如：城市气温、门店销量、API 流量、设备能耗。

**完整代码见仓库根目录的 `demo/` 文件夹**——已经填好所有 prepare 函数和图表，下载就能跑。

## 假设的数据形态

```
列：date, city, country, continent, latitude, longitude, temp_c, weather
```

## 接口设计

- `/api/summary` → `{city_count, record_count, max_temp, min_temp, warming_decade, last_update}`
- `/api/timeline` → 月平均温度 + 12 个月移动平均
- `/api/distribution` → 天气类型占比
- `/api/geo` → 城市经纬度 + 温度（散点用）
- `/api/ranking` → 城市年均温排名（横向条形用）

## 图表（按重要性排序）

1. **月平均气温折线图** + 移动平均线（横跨整行）—— 主图，看长期趋势
2. **城市排名横向条形图** —— 颜色按温度梯度
3. **天气类型环形图**
4. **城市地理散点图** —— 经纬度坐标 + 温度色标，避开 GeoJSON 部署复杂度

## 这个案例独特的地方

- **没用真正的地图组件**（GeoJSON / mapbox），用经纬度散点图代替。原因：部署简单、不依赖外部地图服务、零额外配置
- **十年升温**作为 KPI——用 2014 vs 2023 的年均温对比，体现"会算衍生指标，不只是 max/min"
- 排名图的颜色按温度梯度变化（蓝 → 绿 → 黄 → 红），增加信息密度
